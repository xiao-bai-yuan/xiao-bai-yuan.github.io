Beego框架初识

发表于 2019-09-13 | 更新于: 2019-12-03 | 分类于 Beego框架 |
字数统计: 2.3k字 | 阅读时长 ≈ 14分钟

一、简介

以golang为基础的web开发框架有很多，我挑选了其中的Beego开始学习，它的官网地址如下：

https://beego.me

全中文，对学习使用beego有很大帮助。

beego官网有明确的说明，它有如下特性：

简单化

RESTful 支持、MVC 模型，可以使用 bee 工具快速地开发应用，包括监控代码修改进行热编译、自动化测试代码以及自动化打包部署。

智能化

支持智能路由、智能监控，可以监控 QPS、内存消耗、CPU 使用，以及 goroutine 的运行状况，让您的线上应用尽在掌握。

模块化

beego 内置了强大的模块，包括 Session、缓存操作、日志记录、配置解析、性能监控、上下文操作、ORM 模块、请求模拟等强大的模块，足以支撑你任何的应用。

高性能

beego 采用了 Go 原生的 http 包来处理请求，goroutine 的并发效率足以应付大流量的 Web 应用和 API 应用，目前已经应用于大量高并发的产品中。

MVC，即模型（model）-视图（view）-控制器（controller）的缩写，MVC模式是一种被广泛使用的设计web应用的框架模式，各模块之间相互独立，高度解耦。可参考百度百科了解更详细内容：

https://baike.baidu.com/item/MVC%E6%A1%86%E6%9E%B6/9241230?fromtitle=MVC%E6%A8%A1%E5%BC%8F&fromid=713147&fr=aladdin
