github创建博客

最近想在github上面搞一个自己的博客，找了很多资料，终于搞定，以下主要写一下主要历程以及相关经验，来供大家参考，一起学习，期间参考了很多牛人的博客、视频资料等。非常感谢铺路的前人。

为什么要创建自己的博客

刚开始想搞github博客，就是因为在github上面读代码，方便，然后感觉可以在github上面搞个博客，很装逼。

研究下来发现，其实github上面搭建博客和csdn以及简书上面写自己的博客效果是一样的。最大github上写博客的优势就是，不需要网络，自己也可以写博客，也可以查看自己的博客，搜索自己之前所积累的经验，然后只需要像提交代码一样，可以轻松的放到后台进行保管，以及让大家学习使用。如有问题或者建议，可以加群讨论：

搭建github博客需要学习以及了解的东西

git

本地代码管理工具

下载地址：https://git-scm.com/

github

远程代码管理仓库，也可以用来存储博客内容以及博客的发布

https://github.com/

Node.js

支持hexo以及相关主题的环境

下载地址：https://nodejs.org/zh-cn/download/

hexo

快速、简洁且高效的博客框架

官方地址：https://hexo.io/zh-cn/docs/

NexT

hexo的一种博客的主题，也可以使用其他主题，我学习并使用的是NexT的主题

主要操作地址：http://theme-next.iissnan.com/getting-started.html

markdown

是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式，后缀名为md的文档。

常用的方法以及标签：https://www.runoob.com/markdown/md-tutorial.html

或参考markdown自带的文档：Markdown Reference.md

比较好的一些文章：https://www.jianshu.com/p/399e5a3c7cc5

Typora

是一中md的编辑工具，可以以标签的方式查看文件，也可以直接在上面写自己的博客，来进行生成带标签的md文件。除了这个之外，csdn等也支持markdown的编写。

实际操作

通过了解学习以上的相关知识，我们可以进行github的日志部署以及编写了。

实际操作部分，很多借鉴了：https://www.lixint.me/hexo-blog.html，非常感谢博主，我才能一路学习下来。
